# Автоматизированная система «Анализ дорожной сети» (АС «АДС»)

## 1. О проекте

Автоматизированная система «Анализ дорожной сети» (АС «АДС») предназначена для сбора данных об участках дорожной инфраструктуры на основе нейросетевого анализа спутниковых и наземных снимков. Система автоматизирует процесс извлечения ключевых параметров (ширина проезжей части, количество парковочных мест, тип дорожного покрытия и др.), который традиционно выполняется вручную и требует значительных временных затрат.

### Основные возможности

- **Автоматический сбор данных** — анализ спутниковых снимков и наземных фотографий с использованием нейросетевых моделей.
- **Поддержка видео** — извлечение кадров из видеопотока с последующей обработкой каждого кадра.
- **Улучшение качества изображений** — предобработка медиаматериалов с использованием модели DiffBIR для повышения резкости и устранения шумов.
- **Геометрические измерения** — расчёт реальных размеров объектов с коррекцией перспективы на основе опорных объектов, задаваемых пользователем.
- **Экспорт в GeoJSON** — выгрузка результатов в формате GeoJSON для дальнейшей работы в ГИС-системах (QGIS, PTV Visum).
- **Ролевая модель доступа** — разграничение прав между ролями «Сотрудник» и «Администратор».

### Целевая аудитория

Система ориентирована на сотрудников транспортных лабораторий, проектных организаций и администраций городов, занимающихся транспортным моделированием и планированием городской инфраструктуры.

## 2. Архитектура

Система построена по принципу модульной микро-сервисной архитектуры. Взаимодействие между компонентами осуществляется через REST API и асинхронные очереди задач.

**Ключевые компоненты:**

- **API-сервер (Django + DRF)** — центральный компонент, обрабатывающий HTTP-запросы, управление задачами и аутентификацию пользователей.
- **База данных (PostgreSQL)** — хранение структурированной информации о пользователях, задачах и результатах анализа.
- **Брокер сообщений (Redis)** — обеспечивает асинхронную связь между API-сервером и воркерами Celery.
- **Воркер Celery** — выполняет ресурсоёмкие задачи (предобработка, нейросетевой анализ, геометрические измерения) в фоновом режиме.
- **Объектное хранилище (MinIO)** — S3-совместимое хранилище для медиафайлов (исходные снимки, видео, обработанные изображения).
- **Веб-интерфейс (Vue.js)** — клиентская часть для взаимодействия пользователя с системой.

**Модули анализа:**

- **Наземный анализатор (YOLOv8)** — идентификация и классификация объектов дорожной инфраструктуры на наземных снимках (полосы движения, парковочные места, остановки, здания, тротуары, велодорожки).
- **Спутниковый анализатор (U-Net)** — семантическая сегментация спутниковых снимков для выделения зон (проезжая часть, тротуары, парковки, зелёные зоны, здания).
- **Модуль геометрических измерений (OpenCV)** — расчёт реальных физических размеров объектов с калибровкой по опорному объекту и коррекцией перспективы.
